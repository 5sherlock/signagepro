# SignagePro 개발 일지 (2026-05-07)

## 1. 오늘 완료된 핵심 작업 내역 (미디어 스케줄링 UI 고도화)

오늘 세션에서는 단순한 그룹 단위 방송 시스템에서 벗어나, **개별 기기별 세밀한 타겟팅 및 스케줄링이 가능한 고급 "Swimlane(수평형 타임라인)" UI**로 시스템을 완전히 개편했습니다.

### 1) 직관적인 기기별 수평 타임라인 (Swimlane UI) 도입
- 기존의 복잡했던 '보기 모드' 탭을 제거하고, 각 기기(Device)가 하나의 독립적인 가로줄(트랙)을 갖는 UI로 전면 개편했습니다.
- 관리자는 에셋 라이브러리에서 특정 기기의 트랙으로 직접 파일을 드래그 앤 드롭하여 기기 전용 콘텐츠를 개별적으로 편성할 수 있습니다.
- 기기 트랙 간 드래그 앤 드롭 이동이 자유로워 직관적인 스케줄 관리가 가능해졌습니다.

### 2) 전환 효과(Transition) 및 동기화(Sync) 기능 추가
- 타임라인 내 각 카드마다 개별 재생 시간(초)과 **화면 전환 효과(컷, 디졸브, 밀기, 확대)**를 직접 설정할 수 있는 컨트롤을 추가했습니다.
- 기기 간 재생 타이밍을 똑같이 맞추는 **"재생 시간 동기화(Sync)" 기능**을 추가하여, 개별 콘텐츠를 재생하더라도 여러 화면이 한 몸처럼 움직이게 할 수 있는 기반을 마련했습니다.

### 3) 스마트 상태 비교 로직 (스마트 배포 대기)
- `useMemo`를 활용한 깊은(Deep) 상태 비교 알고리즘을 도입했습니다.
- 파일을 삭제하거나 비웠을 때, 현재 화면 상태가 이미 서버에 저장된 원래 상태와 동일하다면 **불필요한 '(배포 대기)' 마크를 자동으로 없애주는 스마트 로직**을 구현하여 관리자의 혼동을 방지했습니다.

## 2. 향후 나아가야 할 방향 (Next Steps)

오늘 프론트엔드(관리자 대시보드) 측면에서 완벽한 기획과 UI가 완성되었습니다. 다음 세션(다음 날)에서는 이 UI에서 설정한 데이터를 실제로 기기에 반영하는 **백엔드/클라이언트 연동**을 최우선으로 진행해야 합니다.

1. **데이터베이스 및 API 연동 (Backend)**
   - `schema.prisma`의 `PlaylistMedia` 테이블에 추가한 `targetDeviceId` 및 `transition` 필드를 데이터베이스에 반영(`npx prisma db push`).
   - 대시보드에서 `저장 및 기기 배포` 클릭 시 이 개별 설정값들이 백엔드 DB에 올바르게 저장되도록 API 업데이트.
   
2. **클라이언트 플레이어 로직 개편 (Client)**
   - 현장 기기에 설치될 사이니지 클라이언트 앱이 **자신의 Device ID에 해당하는 콘텐츠만 필터링해서 재생**하도록 로직 수정 (병렬 재생 옵션 1 적용).
   - 대시보드에서 설정한 **전환 효과(디졸브 등)**가 클라이언트 웹 화면에서 실제로 렌더링되도록 CSS/JS 트랜지션 로직 추가.
   - (선택 사항) "재생 시간 동기화(Sync)" 옵션 활성화 시 NTP 서버 시간을 기준으로 정확한 초 단위 릴레이를 수행하는 로직 도입.

이 파일을 기준으로 내일 개발을 시작하면 현재의 컨텍스트와 연속성을 끊김 없이 이어나갈 수 있습니다.
