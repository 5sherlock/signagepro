# SignagePro 아키텍처 및 개발 계획 (V2 - 그룹화 모델 적용)

## 1. 시스템 개요
기존 `Y-Board-Premium-Manager`의 1:1 기기 제어 방식을 탈피하여, 5대 규모 이상의 보드를 효율적으로 중앙 관리할 수 있는 **그룹화(Grouping) 기반의 모던 웹 아키텍처(Node.js + React + SQLite)**로 전환합니다. 

## 2. 핵심 기능 및 전략 (기기 그룹화)
단일 기기 조작의 번거로움을 줄이고, 효율성을 극대화하기 위해 기기를 그룹으로 묶어 일괄 제어합니다.
- **일괄 업로드 및 스케줄링**: 영상을 업로드하고 재생 목록을 갱신할 때 개별 기기가 아닌 '그룹 단위'로 명령을 내립니다.
- **하트비트(Heartbeat) 모니터링**: 5대 기기의 온라인/오프라인 상태, CPU/메모리 사용량을 실시간으로 감시하고 시각화합니다.
- **로컬 캐싱**: 네트워크 장애 발생 시에도 기기가 멈추지 않도록 사이니지 보드 로컬에 콘텐츠를 안전하게 보관합니다.

## 3. 아키텍처 상세 설계

### 3.1. 백엔드 서버 (Node.js + Express)
기존의 통신 규격을 유지하면서, 데이터 모델을 그룹 중심으로 개편합니다.
- **데이터베이스 (Prisma + SQLite)**:
  - `Group` 테이블: `id`, `name` (예: 로비, 매장)
  - `Device` 테이블: `id`, `name`, `status`, `groupId` (Group을 참조하는 외래키)
  - `Playlist`/`Media` 테이블: 그룹 단위로 배포될 미디어 스케줄 저장.
- **디바이스 게이트웨이**: 기존 10080 포트를 통한 원시 TCP 소켓 통신을 유지하되, 하트비트 체크 및 그룹 단위의 소켓 브로드캐스팅 로직을 추가합니다.

### 3.2. 관리자 대시보드 (Vite + React)
우수 사이니지(ScreenCloud, MagicINFO)의 장점을 벤치마킹한 직관적이고 세련된 UI를 구축합니다.
- **그룹 관리 패널**: 드래그 앤 드롭으로 기기를 그룹에 할당하고, 그룹별로 미디어를 배포할 수 있는 UI.
- **실시간 관제 그리드**: 5대의 보드 화면을 동시에 모니터링하는 관제실 형태의 대시보드 (Vanilla CSS 기반 모던 다크 모드).

### 3.3. 사이니지 보드 클라이언트 (Electron/React 등)
- 백엔드와 소켓 통신을 유지하며 중앙 서버의 지시(재생, 다운로드, 스크린샷 캡처 등)를 처리.
- (향후 확장) 악보/가사 오버레이 등 고객 특화 기능을 표시할 수 있도록 확장성 있는 컴포넌트 구조로 설계.
