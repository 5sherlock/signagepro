# SignagePro 개발 로드맵 (Development Roadmap)

이 문서는 SignagePro 프로젝트(V2 - 그룹화 모델 적용)의 향후 개발 진행 순서와 단계별 목표를 정의한 마일스톤 계획입니다.

---

## 🚀 Phase 1: 백엔드 기반 및 데이터베이스 모델링
가장 먼저 데이터의 뼈대를 잡고 기기들이 접속할 수 있는 게이트웨이를 구축합니다.
- **1. Prisma DB 스키마 설계**
  - `Group` (그룹), `Device` (기기), `Media` (영상/이미지), `Playlist` (재생 스케줄) 테이블 구조 정의 및 마이그레이션.
- **2. REST API 구축**
  - 대시보드에서 그룹/기기를 등록하고 미디어를 업로드할 수 있는 CRUD API 개발 (`Express`, `Multer` 활용).
- **3. 레거시 단말기 통신용 TCP 소켓 서버 오픈**
  - 기존 10080 포트를 통해 기기들이 접속할 수 있는 TCP 소켓 서버(`net` 모듈) 구축 및 기본 명령어 수신 테스트.

## 💻 Phase 2: 관리자 대시보드 핵심 UI 개발
사용자가 기기를 제어할 수 있는 시각적이고 직관적인 웹 대시보드의 기초를 만듭니다.
- **1. UI/UX 레이아웃 설계**
  - 사이드바(메뉴), 상단바(상태), 메인 뷰로 구성된 레이아웃 작업. (다크 모드, Vanilla CSS 기반)
- **2. 실시간 관제 그리드 (Grid View)**
  - 등록된 5대 이상의 기기 상태(온/오프라인)를 한눈에 볼 수 있는 메인 모니터링 화면 개발.
- **3. 그룹(Group) 및 기기 관리 기능**
  - 기기를 특정 그룹에 할당하고 관리할 수 있는 드래그 앤 드롭 UI 구현.

## 🔄 Phase 3: 실시간 통신 및 미디어 배포 연동
백엔드와 프론트엔드를 연결하고 실시간 제어 로직을 완성합니다.
- **1. 하트비트(Heartbeat) 시스템 연동**
  - 기기 -> 서버(TCP) -> 대시보드(WebSocket)로 이어지는 실시간 상태 변경(CPU/메모리/네트워크 상태) 알림 시스템 구축.
- **2. 그룹 단위 미디어 스케줄링 및 배포**
  - 대시보드에서 특정 그룹에 영상을 할당하면, 서버가 해당 그룹에 속한 모든 기기에게 즉각적으로 다운로드/재생 명령(Socket)을 내리는 로직 구현.

## 🖥️ Phase 4: 사이니지 보드 클라이언트 신규 개발
실제 디스플레이(TV/모니터)에 설치되어 영상을 재생할 단말기용 프로그램을 개발합니다.
- **1. 단말기 아키텍처 구성 (Electron + React)**
  - 전체 화면 모드(Kiosk)로 동작하는 베이스 클라이언트 제작.
- **2. 소켓 통신 및 제어 명령 수신**
  - 서버(10080 포트)에 접속하여 하트비트를 주기적으로 전송하고, 재생/정지/화면 캡처 명령을 수행하는 로직.
- **3. 로컬 캐싱 및 미디어 재생**
  - 서버로부터 미디어 파일을 다운로드하여 로컬에 캐싱하고, 네트워크가 끊겨도 지정된 플레이리스트를 반복 재생하는 안정성 확보 로직.

## ✨ Phase 5: 최적화 및 특화 기능 적용
시스템의 완성도를 높이고, 요구사항에 맞는 특수 기능을 추가합니다.
- **1. 특화 기능 구현**
  - 동영상 위에 악보나 가사를 오버레이(Overlay)하는 커스텀 위젯 적용 (이전 Guitar Copy Master 기능 응용).
- **2. 장애 대비 및 자동 복구 로직**
  - 프로그램 크래시 발생 시 자동 재시작(PM2 또는 Windows Service 연동).
- **3. 최종 테스트 및 배포**
  - 서버 배포(AWS/로컬 PC) 및 보드용 설치 프로그램 패키징(exe 빌드).
