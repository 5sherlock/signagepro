## 3. 미해결 — 내일 우선 과제

### 3.1 U4X+ CM 설치 실패 추적

**현 상태**:
- APK 빌드는 성공 (`android/app/build/outputs/apk/debug/app-debug.apk`)
- 설치 시도 시 "**앱이 설치되지 않았습니다**" 메시지
- "알 수 없는 출처" 허용은 켜져 있음
- 동일 단말에서 어제 다른 APK(`D:\app-debug.apk`, targetSdk 36)는 정상 설치됨

**오늘 시도한 호환성 조정** (전부 적용된 최종 상태):
- `targetSdk 30 → 34`, ABI 필터 `armeabi-v7a`, multidex 라이브러리 명시화
- Material Components 제거, AppCompat만 사용
- `BootReceiver` + `RECEIVE_BOOT_COMPLETED` 제거 (커밋 `ea591a0`)
- `ACCESS_WIFI_STATE`, `FOREGROUND_SERVICE`, `usesCleartextTraffic` 제거
- HOME 카테고리 제거 (LAUNCHER만)

**현재 권한 목록** (최소화 상태):
```
INTERNET
ACCESS_NETWORK_STATE
WAKE_LOCK
```

**비교: 어제 작동한 APK(D:\app-debug.apk)** — `aapt dump`로 확인:
```
패키지명: com.signagepro.player (같음)
targetSdk: 36 (더 높음)
권한: INTERNET 하나
Activity: MainActivity 단 1개
Receiver: 없음
크기: 4.3MB / 4 dex files
서명: V1 + V2 (같음)
```

**남은 가설 (내일 시도 순)**:

1. **Activity 수 줄이기** — 현재 3개(Boot/Setup/Kiosk). 작동 APK는 1개. Setup/Kiosk를 fragment로 합치거나, BootActivity 하나에서 setContentView를 동적으로 바꾸도록.
2. **저장 공간 점검** — 8GB Flash 잔여 용량. 설정 → 저장공간에서 사용 가능 MB 확인. 9.4MB APK + 설치 후 확장 → 30MB 정도 필요.
3. **"앱 확인(Verify apps)" 비활성화** — 설정 → 보안 → "앱 확인" OFF. 일부 셋톱 ROM이 외부 검증 시도하다 실패.
4. **이전 패키지 잔재** — 설정 → 앱에서 "SignagePro" 또는 "com.signagepro" 검색해서 잔재 제거.
5. **APK signature scheme 강제 V1만** — 일부 구형 ROM이 V2 메타데이터 오해 가능. `signingConfigs.debug { enableV2Signing = false }` 시도.
6. **의존성 추가 축소** — Retrofit/Moshi 제거하고 OkHttp + org.json만으로 재작성 → dex 4~5개 수준까지.

**ADB 연결 확보**가 가장 큰 진전. USB 케이블로 한 번만 연결되면 `adb install`로 즉각 실패 사유(`INSTALL_FAILED_*`)를 알 수 있어 위 가설들을 한 줄로 검증 가능. 그러나 사용자가 USB 디버깅을 켰음에도 단말 USB 포트 상황은 미확인.

### 3.2 자동 부팅 기능 후속 처리

`BootReceiver` 제거로 단말 재부팅 시 앱 자동 실행이 안 됨. 후속 옵션:
- 정식 launcher 등록 (HOME 카테고리 추가) — 일부 ROM만 허용
- 외부 부팅 트리거 앱 (Tasker 등)
- ROM 자체 설정의 "부팅 시 자동 실행" 메뉴 사용

설치 문제 해결 후 결정.

### 3.3 5대 동기 재생 현장 검증

설치 성공해도 다음 검증 미완:
- NTP 동기 후 실제 5대가 같은 슬라이드 표시하는지
- 서버 다운 상태에서 캐시된 playlist로 계속 재생되는지
- 네트워크 끊김 복구 후 재연결되는지
